Instrucciones de Uso

En esta página encontrarás una guía detallada para descargar, ejecutar y eliminar las máquinas de DockerLabs, así como una sección de solución de errores. Estas instrucciones están basadas en la guía oficial de DockerLabs contenida en el PDF incluido en la plataforma.

Requisitos Previos

Para que las máquinas de DockerLabs puedan funcionar, es necesario tener Docker instalado. 

sudo apt install docker.io

Cómo ejecutar las máquinas

Una vez descargues una máquina en formato .tar, verás que incluye un script llamado auto_deploy.sh. Solo tienes que ejecutarlo para desplegar o eliminar la máquina. 

sudo bash auto_deploy.sh laboratorio.tar

Cómo ejecutar las máquinas de Pivoting

Si has descargado una máquina de pivoting y la has descomprimido, verás varios archivos .tar etiquetados como:

  • máquina1.tar
  • máquina2.tar
  • máquina3.tar
  • máquina4.tar

Junto a estos archivos encontrarás el script auto_deploy.sh, encargado de montar la red de pivoting automáticamente. 

sudo bash auto_deploy.sh máquina1.tar máquina2.tar máquina3.tar máquina4.tar

Cómo eliminar las máquinas

Para eliminar cualquier máquina, simplemente presiona: 

Ctrl + C

Esto borrará automáticamente los contenedores y redes creadas por el laboratorio.

Mensaje de confirmación en máquinas de Pivoting

En las máquinas de pivoting, al eliminarlas se te hará una pregunta adicional: 

¿Quieres eliminar también todas las imágenes de Docker? (Si|No)

Si no te preocupa perder imágenes que ya tengas en tu sistema, selecciona Sí. Si tienes imágenes que deseas conservar, responde No y podrás eliminarlas manualmente más tarde.
